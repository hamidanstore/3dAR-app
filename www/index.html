Langkah-Langkah Menyiapkan GitHub untuk AR Hosting

Langkah 1: Buat Akun & Repository Baru

  1. Buka github.com dan buat akun gratis jika belum punya.
  2. Setelah login, klik tombol "New" (berwarna hijau) di sebelah kiri untuk membuat Repository baru.
  3. Beri nama repository (contoh: katalog-ar).
  4. Pastikan kamu memilih mode Public (Sangat penting agar file 3D bisa dibaca oleh browser HP).
  5. Centang opsi "Add a README file", lalu klik Create repository.

Langkah 2: Aktifkan GitHub Pages (Web Hostingnya)

  1. Di dalam repository katalog-ar yang baru dibuat, klik tab "Settings" (ikon gerigi di atas).
  2. Di menu sebelah kiri, cari dan klik menu "Pages".
  3. Pada bagian Build and deployment -> Branch, ubah tulisan "None" menjadi "main".
  4. Klik Save.
  5. Tunggu sekitar 1-2 menit. Jika kamu me-refresh halaman itu, akan muncul tulisan "Your site is live at https://[username-mu].github.io/katalog-ar/". Ini akan menjadi dasar link produkmu nanti!

Langkah 3: Buat Token Rahasia (Personal Access Token)

Token ini adalah "kunci" agar koding-mu bisa mengunggah file otomatis ke GitHub tanpa perlu login manual.

  1. Klik foto profilmu di pojok kanan atas GitHub, lalu pilih "Settings".
  2. Scroll ke paling bawah di menu kiri, klik "Developer settings".
  3. Klik "Personal access tokens", lalu pilih "Tokens (classic)".
  4. Klik tombol "Generate new token (classic)" di kanan atas.
  5. Di kolom Note, isi dengan nama aplikasimu (misal: "Aplikasi AR Hamidan").
  6. Di bagian Expiration, ubah menjadi "No expiration" agar tokennya tidak kadaluarsa.
  7. Di bagian Select scopes (kotak-kotak centang), centang kotak repo (ini memberikan akses penuh untuk membuat dan menghapus file).
  8. Scroll ke bawah dan klik "Generate token".
  9. PENTING: Akan muncul deretan kode panjang yang diawali dengan ghp_.... Salin dan simpan kode ini di tempat yang aman (misal di Notepad/WhatsApp). Kode ini hanya ditampilkan sekali! Jika hilang, kamu harus membuat yang baru.
